属性注册：__setattr__ 自动分类路由

当执行 self.conv = nn.Conv2d(3, 64, 3) 时，__setattr__ 如何自动检测赋值类型， 并将其路由到 _modules。同样地，nn.Parameter → _parameters， 已注册 Buffer → _buffers，普通属性 → object.__setattr__。 这是 Module 系统最核心的魔法。

Forward 调用链：model(x) 到底发生了什么

永远不要调用 model.forward(x)，要调用 model(x)。 因为 __call__ 会在 forward 前后执行 hook 系统。 执行顺序：全局 pre-hooks → 局部 pre-hooks → forward → 局部 hooks → 全局 hooks。 每个 hook 都可以修改输入/输出，甚至替换 forward 的返回值。

设备与类型转换：_apply 是所有转换的引擎

model.cuda()、model.float()、model.to(device) 等全部委托给 _apply(fn)。 关键顺序：先递归处理子模块 → 再转换自己的参数（含 .grad）→ 最后转换自己的 Buffer。 而 model.apply(fn)（少了下划线）走的是另一条路——对每个子模块递归调用 fn(module)，用于权重初始化。

状态序列化：模型保存与加载的完整流程

state_dict() 递归收集所有持久化参数和 Buffer（跳过 persistent=False 的 Buffer）， 返回以层级路径为 key 的 OrderedDict。load_state_dict() 反向操作， 返回 _IncompatibleKeys(missing_keys, unexpected_keys)。 Hook 系统允许在保存/加载前后注入自定义逻辑。

层级导航：Module 树的 DFS 遍历与去重

named_modules() 使用 DFS pre-order 遍历整个 Module 树，通过 memo set 确保每个 Module 只被 yield 一次。named_parameters() 和 named_buffers() 共用 _named_members() 引擎——先遍历 Module 树，再从每个 Module 中提取对应类型的成员。 children() 只返回直接子模块（非递归）。
