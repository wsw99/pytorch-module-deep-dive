nn.Module 的核心能力之一是层级化管理子模块。PyTorch 提供了五种容器类，让你可以用 list、dict、sequential 等数据结构来组织子模块和参数。本 Notebook 深入这些容器类的源码，揭示它们与 Python 原生数据结构的关键差异。

A.1 Sequential — 链式调用的有序容器

Sequential 是最常用的容器类。它的 forward() 方法实现了链式调用：将前一个模块的输出作为下一个模块的输入。

In [2]:
# Sequential 核心源码
from torch.nn.modules import container as cont

print("=== Sequential.__init__ ===")
print(inspect.getsource(cont.Sequential.__init__))
print("\n=== Sequential.forward ===")
print(inspect.getsource(cont.Sequential.forward))
print("\n=== Sequential.__getitem__（索引/切片）===")
print(inspect.getsource(cont.Sequential.__getitem__))

=== Sequential.__init__ ===
    @overload
    def __init__(self, *args: Module) -> None: ...

    @overload
    def __init__(self, arg: OrderedDict[str, Module]) -> None: ...

    def __init__(self, *args):
        super().__init__()
        if len(args) == 1 and isinstance(args[0], OrderedDict):
            for key, module in args[0].items():
                self.add_module(key, module)
        else:
            for idx, module in enumerate(args):
                self.add_module(str(idx), module)


=== Sequential.forward ===
    def forward(self, input):
        for module in self:
            input = module(input)
        return input


=== Sequential.__getitem__（索引/切片）===
    @_copy_to_script_wrapper
    def __getitem__(self, idx: Union[slice, int]) -> Union[Sequential, Module]:
        if isinstance(idx, slice):
            return self.__class__(OrderedDict(list(self._modules.items())[idx]))
        else:
            return self._get_item_by_idx(self._modules.values(), idx)

Sequential.__init__ 的两种构造方式：

构造方式 示例 内部行为
位置参数 Sequential(conv, relu, fc) 自动编号为 "0", "1", "2"
OrderedDict Sequential(OrderedDict([("conv", conv), ("relu", relu)])) 使用指定的 key 名称

__getitem__ 支持两种访问：

  • 整数索引 → 返回单个 Module（支持负数索引 [-1]）
  • 切片 → 返回新的 Sequential（保留 key 的 OrderedDict）

=== Sequential.append ===
    def append(self, module: Module) -> Self:
        r"""Append a given module to the end."""
        self.add_module(str(len(self)), module)
        return self


=== Sequential.__setitem__（按索引替换）===
    def __setitem__(self, idx: int, module: Module) -> None:
        key: str = self._get_item_by_idx(self._modules.keys(), idx)
        return setattr(self, key, module)


=== Sequential.__delitem__（按索引/切片删除）===
    def __delitem__(self, idx: Union[slice, int]) -> None:
        if isinstance(idx, slice):
            for key in list(self._modules.keys())[idx]:
                delattr(self, key)
        else:
            key = self._get_item_by_idx(self._modules.keys(), idx)
            delattr(self, key)
        # To preserve numbering
        str_indices = [str(i) for i in range(len(self._modules))]
        self._modules = OrderedDict(list(zip(str_indices, self._modules.values())))


=== Sequential.insert ===
    def insert(self, index: int, module: Module) -> Self:
        if not isinstance(module, Module):
            raise AssertionError(f"module should be of type: {Module}")
        n = len(self._modules)
        if not (-n <= index <= n):
            raise IndexError(f"Index out of range: {index}")
        if index < 0:
            index += n
        for i in range(n, index, -1):
            self._modules[str(i)] = self._modules[str(i - 1)]
        self._modules[str(index)] = module
        return self
💡 关键细节：__delitem__ 删除元素后会重新编号所有 key（"0", "1", "2"...），保证编号始终连续。insert 也会将后续元素逐一后移以保持连续性。

A.2 ModuleList — 可注册的模块列表

ModuleList 看起来像 Python list，但内部使用 OrderedDict 存储子模块（与 Sequential 相同），因此所有子模块会被 parameters()、modules() 等方法自动发现。

In [4]:
# ModuleList 核心源码
print("=== ModuleList.__init__ ===")
print(inspect.getsource(cont.ModuleList.__init__))
print("\n=== ModuleList.__getitem__ ===")
print(inspect.getsource(cont.ModuleList.__getitem__))
print("\n=== ModuleList.append ===")
print(inspect.getsource(cont.ModuleList.append))
print("\n=== ModuleList.extend ===")
print(inspect.getsource(cont.ModuleList.extend))
print("\n=== ModuleList.insert ===")
print(inspect.getsource(cont.ModuleList.insert))

=== ModuleList.__init__ ===
    def __init__(self, modules: Optional[Iterable[Module]] = None) -> None:
        super().__init__()
        if modules is not None:
            self += modules


=== ModuleList.__getitem__ ===
    @overload
    def __getitem__(self, idx: slice) -> ModuleList: ...

    @overload
    def __getitem__(self, idx: int) -> Module: ...

    @_copy_to_script_wrapper
    def __getitem__(self, idx: Union[int, slice]) -> Union[Module, ModuleList]:
        if isinstance(idx, slice):
            return self.__class__(list(self._modules.values())[idx])
        else:
            return self._modules[self._get_abs_string_index(idx)]


=== ModuleList.append ===
    def append(self, module: Module) -> Self:
        r"""Append a given module to the end of the list."""
        self.add_module(str(len(self)), module)
        return self


=== ModuleList.extend ===
    def extend(self, modules: Iterable[Module]) -> Self:
        r"""Append modules from a Python iterable to the end of the list."""
        if not isinstance(modules, container_abcs.Iterable):
            raise TypeError(
                "ModuleList.extend should be called with an "
                "iterable, but got " + type(modules).__name__
            )
        offset = len(self)
        for i, module in enumerate(modules):
            self.add_module(str(offset + i), module)
        return self


=== ModuleList.insert ===
    def insert(self, index: int, module: Module) -> None:
        r"""Insert a given module before a given index in the list."""
        for i in range(len(self._modules), index, -1):
            self._modules[str(i)] = self._modules[str(i - 1)]
        self._modules[str(index)] = module
⚠️ 关键发现：ModuleList 没有 forward() 方法（继承 Module 的默认实现会抛 NotImplementedError）。它只是一个存储容器，而非计算单元。你需要手动迭代它来定义 forward 逻辑。

ModuleList.__repr__ 还做了一个聪明的优化：相邻重复模块会压缩显示（如 (0-9): 10 x Linear(...)）。

A.3 ModuleDict — 基于 key 的子模块字典

In [5]:
# ModuleDict 核心源码
print("=== ModuleDict.__init__ ===")
print(inspect.getsource(cont.ModuleDict.__init__))
print("\n=== ModuleDict.__getitem__ / __setitem__ / __delitem__ ===")
print(inspect.getsource(cont.ModuleDict.__getitem__))
print(inspect.getsource(cont.ModuleDict.__setitem__))
print(inspect.getsource(cont.ModuleDict.__delitem__))
print("\n=== ModuleDict.update ===")
print(inspect.getsource(cont.ModuleDict.update))

=== ModuleDict.__init__ ===
    def __init__(self, modules: Optional[Mapping[str, Module]] = None) -> None:
        super().__init__()
        if modules is not None:
            self.update(modules)


=== ModuleDict.__getitem__ / __setitem__ / __delitem__ ===
    @_copy_to_script_wrapper
    def __getitem__(self, key: str) -> Module:
        return self._modules[key]

    def __setitem__(self, key: str, module: Module) -> None:
        self.add_module(key, module)

    def __delitem__(self, key: str) -> None:
        del self._modules[key]


=== ModuleDict.update ===
    def update(self, modules: Mapping[str, Module]) -> None:
        if not isinstance(modules, container_abcs.Iterable):
            raise TypeError(
                "ModuleDict.update should be called with an "
                "iterable of key/value pairs, but got " + type(modules).__name__
            )

        if isinstance(modules, (OrderedDict, ModuleDict, container_abcs.Mapping)):
            for key, module in modules.items():
                self[key] = module
        else:
            # modules here can be a list with two items
            for j, m in enumerate(modules):
                if not isinstance(m, container_abcs.Iterable):
                    raise TypeError(
                        "ModuleDict update sequence element "
                        "#" + str(j) + " should be Iterable; is" + type(m).__name__
                    )
                if not len(m) == 2:
                    raise ValueError(
                        "ModuleDict update sequence element "
                        "#" + str(j) + " has length " + str(len(m)) + "; 2 is required"
                    )
                self[m[0]] = m[1]  # type: ignore[assignment]

ModuleDict 的设计要点：

  • __setitem__ 直接调用 add_module(key, module)，所以 module_dict["conv"] = nn.Conv2d(...) 就能正确注册子模块
  • __delitem__ 直接从 _modules 中删除（无需重编号，因为是命名 key）
  • update() 支持三种输入：OrderedDict、普通 dict/Mapping、以及 [(key, module), ...] 列表

A.4 ParameterList / ParameterDict — 参数容器

这两个类专门管理参数（而非子模块），核心功能是将普通 Tensor自动包装为 nn.Parameter：

In [6]:
# ParameterList / ParameterDict 核心源码
print("=== ParameterList.__setitem__（自动包装 Tensor → Parameter）===")
print(inspect.getsource(cont.ParameterList.__setitem__))
print("\n=== ParameterList.append ===")
print(inspect.getsource(cont.ParameterList.append))
print("\n=== ParameterDict.__setitem__（自动包装）===")
print(inspect.getsource(cont.ParameterDict.__setitem__))

=== ParameterList.__setitem__（自动包装 Tensor → Parameter）===
    def __setitem__(self, idx: int, param: Any) -> None:
        # Note that all other function that add an entry to the list part of
        # the ParameterList end up here. So this is the only place where we need
        # to wrap things into Parameter if needed.
        # Objects added via setattr() are not in the list part and thus won't
        # call into this function.
        idx = self._get_abs_string_index(idx)
        if isinstance(param, torch.Tensor) and not isinstance(param, Parameter):
            param = Parameter(param)
        return setattr(self, str(idx), param)


=== ParameterList.append ===
    def append(self, value: Any) -> Self:
        """Append a given value at the end of the list."""
        new_idx = len(self)
        self._size += 1
        self[new_idx] = value
        return self


=== ParameterDict.__setitem__（自动包装）===
    def __setitem__(self, key: str, value: Any) -> None:
        self._keys[key] = None
        attr = self._key_to_attr(key)
        if isinstance(value, torch.Tensor) and not isinstance(value, Parameter):
            value = Parameter(value)
        setattr(self, attr, value)
⚠️ 重要区别：ParameterList.__setitem__ 是唯一进行 Tensor→Parameter 包装的地方。所有增删操作（append, extend, insert, __iadd__）最终都会调用 __setitem__。但直接通过 setattr(self, name, tensor) 设置的属性不会走 __setitem__，因此不会被自动包装。

B.3 ModuleList vs Python list：参数可见性的核心差异

这是最容易踩的坑。Python 原生 list 中的 Module 不会被 parameters() 发现：

┌──────────────────────────────────────────────────────────┐
│              Python list (错误做法)                       │
│                                                          │
│  self.layers = [nn.Linear(10,10) for _ in range(3)]     │
│                                                          │
│  model.parameters()  →  ❌ 空的！                        │
│  model.to('cuda')    →  ❌ 子模块不会迁移！              │
│  state_dict()        →  ❌ 子模块参数不会保存！          │
│                                                          │
│  原因：Python list 不会触发 __setattr__ 拦截             │
└──────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────┐
│              nn.ModuleList (正确做法)                     │
│                                                          │
│  self.layers = nn.ModuleList([                           │
│      nn.Linear(10,10) for _ in range(3)                  │
│  ])                                                      │
│                                                          │
│  model.parameters()  →  ✅ 所有子模块参数都可见          │
│  model.to('cuda')    →  ✅ 所有子模块一起迁移            │
│  state_dict()        →  ✅ 所有子模块参数都保存          │
│                                                          │
│  原因：ModuleList 通过 add_module() 注册子模块           │
└──────────────────────────────────────────────────────────┘

B.4 容器类的 state_dict 命名规则

容器类在 state_dict() 中的 key 命名取决于它们在父模块中的属性名以及容器类型：

容器类型 state_dict key 格式示例 说明
Sequential (自动编号) features.0.weight 数字 key 作为路径段
Sequential (OrderedDict) features.conv.weight 命名 key 作为路径段
ModuleList layers.0.weight, layers.1.bias 始终是数字索引
ModuleDict choices.conv.weight, choices.pool.weight 字符串 key
ParameterList params.0 (直接是 Parameter) 数字索引，无子属性
ParameterDict embeddings.word_emb 字符串 key

这个命名规则自然地保持了层级结构，与 _modules / _parameters 的键名一一对应。

实验 1：ModuleList vs Python list — 参数可见性对比

In [8]:
# 构建两个结构相同但容器不同的模型
class ModelWithList(nn.Module):
    """使用 Python 原生 list（❌ 错误做法）"""
    def __init__(self):
        super().__init__()
        self.layers = [nn.Linear(10, 10) for _ in range(3)]

    def forward(self, x):
        for layer in self.layers:
            x = layer(x)
        return x

class ModelWithModuleList(nn.Module):
    """使用 nn.ModuleList（✅ 正确做法）"""
    def __init__(self):
        super().__init__()
        self.layers = nn.ModuleList([nn.Linear(10, 10) for _ in range(3)])

    def forward(self, x):
        for layer in self.layers:
            x = layer(x)
        return x

# 创建两个模型

model_modlist = ModelWithModuleList()

实验 3：ModuleDict 动态增删子模块

# ModuleDict 的动态管理能力
class RouterModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.experts = nn.ModuleDict({
            'linear': nn.Linear(10, 10),
            'conv': nn.Conv1d(1, 1, 3, padding=1),
        })

    def forward(self, x, expert_name):
        return self.experts[expert_name](x)

model = RouterModel()
print("初始 experts:")
for name, mod in model.experts.items():
    print(f"  {name}: {mod}")
print(f"experts keys: {list(model.experts.keys())}")

# --- 动态添加 ---
print(f"\n=== 动态添加 expert ===")
model.experts['mlp'] = nn.Sequential(
    nn.Linear(10, 20),
    nn.ReLU(),
    nn.Linear(20, 10)
)
print(f"experts keys: {list(model.experts.keys())}")

# 验证新模块被正确注册
print(f"mlp 是否在 named_modules 中: {'experts.mlp' in dict(model.named_modules())}")

# --- 动态替换 ---
print(f"\n=== 动态替换 ===")
print(f"替换前 linear: {model.experts['linear']}")
model.experts['linear'] = nn.Linear(10, 10, bias=False)  # 换一个无 bias 版本
print(f"替换后 linear: {model.experts['linear']}")

# --- 动态删除 ---
print(f"\n=== 动态删除 ===")
del model.experts['conv']
print(f"删除 conv 后 keys: {list(model.experts.keys())}")
print(f"conv 是否还在 named_modules 中: {'experts.conv' in dict(model.named_modules())}")

# --- pop 操作 ---
print(f"\n=== pop 操作 ===")
popped = model.experts.pop('mlp')
print(f"pop mlp: {popped}")
print(f"剩余 keys: {list(model.experts.keys())}")

# --- update 操作 ---
print(f"\n=== update 批量添加 ===")
model.experts.update({
    'norm': nn.LayerNorm(10),
    'dropout': nn.Dropout(0.3),
})
print(f"update 后 keys: {list(model.experts.keys())}")
print(f"参数数量: {sum(1 for _ in model.parameters())}")

# --- 正向传播 ---
x = torch.randn(4, 10)
out = model(x, 'linear')
print(f"\nforward(x, 'linear') output shape: {out.shape}")

Part D: 小结

容器类选择决策树

需要组织子模块？
│
├── 模块之间是串联计算关系？
│   └── Sequential ✅
│       ├── 简单链式：Sequential(mod1, mod2, mod3)
│       └── 需要命名：Sequential(OrderedDict([...]))
│
├── 需要按索引访问，forward 中手动迭代？
│   └── ModuleList ✅
│       └── 永远不要用 Python list 存 Module！
│
├── 需要按名称动态路由？
│   └── ModuleDict ✅
│       └── 适合 MoE、条件计算等场景
│
└── 需要组织参数（非子模块）？
    ├── 按索引访问 → ParameterList ✅
    └── 按名称访问 → ParameterDict ✅

要点清单

  • Sequential 的 forward() 实现链式调用：for module in self: input = module(input)
  • Sequential.__getitem__ 支持整数索引（返回 Module）和切片（返回新 Sequential）
  • Sequential 和 ModuleList 在删除/插入元素后会重新编号
  • ModuleList 没有 forward() 方法——它是存储容器，不是计算单元
  • 永远不要用 Python list 存储 Module：parameters() 和 to() 都不会递归进去
  • ModuleDict 通过 __setitem__ → add_module() 实现自动注册
  • ParameterList.__setitem__ 是唯一的 Tensor→Parameter 自动包装入口
  • 容器类的 state_dict key 由属性名 + 内部 key 组成，层级结构与 _modules 一致

常见陷阱

陷阱 说明
Python list 存 Module parameters() 为空，to(device) 无效，state_dict 不保存
Sequential 切片保留原始 key seq[1:3] 返回的 Sequential 保留了 "1" 和 "2" 的 key（不是从 0 重新编号）
ModuleList forward 未定义 调用 ModuleList()(x) 会抛 NotImplementedError
ParameterList 不能直接迭代 __iter__ 使用 self[i] 访问，O(n²) 复杂度，应避免频繁迭代
del ModuleDict item 后仍可被引用 已从 _modules 删除，但如果是通过 pop 操作持有引用，模块仍然存活
