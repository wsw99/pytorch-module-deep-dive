A.1 state_dict — 导出模型完整状态

state_dict() 是 PyTorch 模型保存/加载的基石。它会递归遍历所有子模块，将每个模块的 _parameters 和 persistent _buffers 收集到一个 OrderedDict 中。

核心调用链：state_dict() → _save_to_state_dict()（递归实现）

先看一下 state_dict 的完整源码：

In [2]:

=== state_dict 源码 ===
    def state_dict(self, *args, destination=None, prefix="", keep_vars=False):
        r"""Return a dictionary containing references to the whole state of the module.

        Both parameters and persistent buffers (e.g. running averages) are
        included. Keys are corresponding parameter and buffer names.
        When keep_vars is ``True``, the returned dictionary contains references
        to the original tensors instead of detached copies.

        .. warning::
            Currently, ``state_dict()`` also accepts positional arguments for
            ``destination``, ``prefix`` and ``keep_vars`` in order. However,
            this is being deprecated and keyword arguments will be enforced in
            a future release.

        .. warning::
            Setting ``keep_vars=True`` should be used cautiously: the resulting
            dictionary will keep references to the original tensors, not copies.
            Modifying the tensor values after calling ``state_dict(keep_vars=True)``
            will be reflected in the original module's state, and vice versa.

        .. warning::
            Please avoid the use of argument ``destination`` as it is not
            designed for end-users.

        Args:
            destination (dict, optional): Deprecated. This dict will be updated
                with the module's state_dict. Use the returned dict instead.
            prefix (str, optional): Deprecated. A prefix added to parameter
                and buffer names to compose the keys in state_dict. Default: ``''``.
            keep_vars (bool, optional): Deprecated. If True, returns references
                to the original tensors. Default: ``False``.

        Returns:
            dict:
                a dictionary containing a whole state of the module

        Example::

            >>> # xdoctest: +SKIP("undefined vars")
            >>> module.state_dict()

        """
        # TODO: Remove `args` and the parsing logic when BC allows.
        if len(args) > 3:
            raise TypeError(
                f"state_dict() takes at most 3 arguments ({len(args)} given)"
            )
        parsed_args: dict[str, Any] = {
            "destination": destination,
            "prefix": prefix,
            "keep_vars": keep_vars,
        }
        arg_names = ("destination", "prefix", "keep_vars")
        for i, arg in enumerate(args):
            if arg is not None or arg_names[i] == "keep_vars":
                parsed_args[arg_names[i]] = arg
        destination = parsed_args["destination"]
        prefix = parsed_args["prefix"]
        keep_vars = parsed_args["keep_vars"]

        if destination is None:
            destination = OrderedDict()
            destination._metadata = OrderedDict()

        destination._metadata[prefix[:-1]] = local_metadata = dict(version=self._version)
        self._save_to_state_dict(destination, prefix, keep_vars)
        for name, module in self._modules.items():
            if module is not None:
                module.state_dict(
                    destination=destination, prefix=prefix + name + ".", keep_vars=keep_vars
                )
        for hook in self._state_dict_hooks.values():
            hook_result = hook(self, destination, prefix, local_metadata)
            if hook_result is not None:
                destination = hook_result
        return destination

state_dict() 关键流程分析：

state_dict(destination, prefix, keep_vars)
│
├── 1. 初始化 destination OrderedDict（带 _metadata）
├── 2. _save_to_state_dict(destination, prefix, keep_vars)
│      └── 将当前模块的 _parameters + persistent _buffers 写入 destination
├── 3. 递归：对每个子模块调用 module.state_dict(destination, prefix + name + ".", ...)
│      └── 子模块的参数会被加上层级前缀（如 "conv.weight", "conv.bias"）
├── 4. 执行 _state_dict_hooks（后处理 hook）
└── 返回 destination

=== _save_to_state_dict 源码 ===
    def _save_to_state_dict(self, destination, prefix, keep_vars):
        r"""Save module state to the ``destination`` dictionary.

        The ``destination`` dictionary will contain the state
        of the module, but not its descendants. This is called on every
        submodule in :meth:`~torch.nn.Module.state_dict`.

        In rare cases, subclasses can achieve custom behavior by
        overriding this method. Most users should use
        :meth:`~torch.nn.Module.state_dict` instead.

        Args:
            destination (dict): a dict where state will be stored
            prefix (str): the prefix for parameters and buffers used in this
                module
            keep_vars (bool): if ``True``, this method returns references to
                the original tensors. If ``False``, detached copies.
        """
        for name, param in self._parameters.items():
            if param is not None:
                destination[prefix + name] = param if keep_vars else param.detach()
        for name, buf in self._buffers.items():
            if buf is not None and name not in self._non_persistent_buffers_set:
                destination[prefix + name] = buf if keep_vars else buf.detach()
        # Populate any extra state.
        extra_state_key = prefix + _EXTRA_STATE_KEY_SUFFIX
        destination[extra_state_key] = self.get_extra_state()
        if isinstance(destination[extra_state_key], dict):
            destination._metadata[prefix[:-1]][_EXTRA_STATE_KEY_SUFFIX] = (
                destination[extra_state_key].copy()
            )

关键要点：

要点 说明
persistent buffer 过滤 _non_persistent_buffers_set 中的 buffer 不会被保存（如 Dropout 的 mask）
keep_vars=False（默认） 返回参数/缓冲区的 detached 副本，修改返回值不会影响模型
keep_vars=True 返回原始 tensor 引用，节省内存但需谨慎使用
extra_state 自动调用 get_extra_state() 保存额外自定义状态

A.2 load_state_dict — 加载模型状态

load_state_dict() 的接口看似简单，内部实现却很精细。它支持：

  • strict=True（默认）：要求 state_dict 中的 key 与模型完全匹配
  • strict=False：允许 key 不匹配，返回 _IncompatibleKeys 告知哪些 key 缺失/多余
  • assign=False（默认）：将 state_dict 中的 tensor copy 到模型参数中
  • assign=True：将模型参数的 data 替换为 state_dict 中的 tensor（用于分布式场景）
In [4]:
print("=== load_state_dict 源码 ===")
print(inspect.getsource(nn.Module.load_state_dict))

=== load_state_dict 源码 ===
    def load_state_dict(self, state_dict, strict=True, assign=False):
        r"""Copy parameters and buffers from :attr:`state_dict` into this module and its descendants.

        If :attr:`strict` is ``True``, then the keys of :attr:`state_dict` must
        exactly match the keys returned by this module's
        :meth:`~torch.nn.Module.state_dict` function.

        .. warning::
            When ``assign=True``, the ``state_dict`` tensors must share the same
            underlying storage with the model's parameters (e.g., both on the
            same device). This is typically used when loading state from a
            parallel checkpoint into a model with ``torch.distributed``.

        Args:
            state_dict (dict): a dict containing parameters and persistent buffers.
            strict (bool, optional): whether to strictly enforce that the keys
                in :attr:`state_dict` match the keys returned by this module's
                :meth:`~torch.nn.Module.state_dict` function. Default: ``True``
            assign (bool, optional): whether to assign items in the state
                dictionary to their corresponding keys in the module instead
                of copying them inplace into the module's current parameters and buffers.
                When ``False``, the properties of the tensors in the current
                module are preserved while when ``True``, the properties of the
                Tensors in the state dict are preserved. The only
                exception is the ``requires_grad`` field of
                :class:`~torch.nn.Parameter` for which the value from the
                module is preserved.
                Default: ``False``

        Returns:
            ``NamedTuple`` with ``missing_keys`` and ``unexpected_keys`` fields:
                * **missing_keys** is a list of str containing the missing keys
                * **unexpected_keys** is a list of str containing the unexpected keys

        Note:
            If a parameter or buffer is registered as ``None`` and its
            corresponding key exists in :attr:`state_dict`, it will
            not be loaded.
        """
        missing_keys: list[str] = []
        unexpected_keys: list[str] = []
        error_msgs: list[str] = []

        # copy state_dict so _load_from_state_dict can modify it
        metadata = getattr(state_dict, "_metadata", None)
        state_dict = OrderedDict(state_dict)
        if metadata is not None:
            # 如果原 state_dict 有 _metadata，保留一个副本
            state_dict._metadata = metadata

        def load(module, local_state_dict, prefix=""):
            local_metadata = {} if metadata is None else metadata.get(prefix[:-1], {})
            if self._version != local_metadata.get("version", None) and strict:
                error_msgs.append(
                    f"Unexpected version number in state_dict for {prefix[:-1]}: "
                    f"expected version {self._version}, got {local_metadata.get('version', None)}"
                )
            args = (
                state_dict,
                prefix,
                local_metadata,
                True,  # strict
                missing_keys,
                unexpected_keys,
                error_msgs,
            )
            # 注意：_load_from_state_dict 的签名中有 strict 参数，
            # 用于逐 key 检查（而非全局 strict）
            if is_full_backward_hook is not None or not module._has_load_state_dict_hook:
                module._load_from_state_dict(*args)
            # 执行 load_state_dict pre-hook
            for hook in module._load_state_dict_pre_hooks.values():
                hook(
                    state_dict,
                    prefix,
                    local_metadata,
                    strict,
                    missing_keys,
                    unexpected_keys,
                    error_msgs,
                )
            # 递归加载子模块
            for name, child in module._modules.items():
                if child is not None:
                    child_prefix = prefix + name + "."
                    child_state_dict = {
                        k: v
                        for k, v in state_dict.items()
                        if k.startswith(child_prefix)
                    }
                    load(child, child_state_dict, child_prefix)  # 注意这里传入的是 filtered child_state_dict

            # 注意：这里不再用 local_state_dict 去加载子模块，
            # 而是在 load() 内部过滤后传给子模块
            # PyTorch 2.8 做了这个优化

        load(self, state_dict)
        # 删除 load 函数，解除循环引用
        del load

        # strict 模式下的严格检查
        if strict:
            if len(unexpected_keys) > 0:
                error_msgs.insert(
                    0,
                    f"Unexpected key(s) in state_dict: {', '.join(repr(k) for k in unexpected_keys)}",
                )
            if len(missing_keys) > 0:
                error_msgs.insert(
                    0,
                    f"Missing key(s) in state_dict: {', '.join(repr(k) for k in missing_keys)}",
                )

        if len(error_msgs) > 0:
            raise RuntimeError(
                "Error(s) in loading state_dict for {}:\n\t{}".format(
                    self.__class__.__name__, "\n\t".join(error_msgs)
                )
            )
        return _IncompatibleKeys(missing_keys, unexpected_keys)

load_state_dict 执行流程图：

load_state_dict(state_dict, strict=True)
│
├── 1. 将 state_dict 转为 OrderedDict（保留 _metadata）
├── 2. 定义递归函数 load(module, prefix)
│      │
│      ├── 2a. 调用 module._load_from_state_dict()
│      │       └── 对每个 _parameters[name]，从 state_dict 中 copy 数据
│      │       └── 对每个 _buffers[name]（persistent），从 state_dict 中 copy 数据
│      │       └── 检查 missing_keys / unexpected_keys
│      │
│      ├── 2b. 执行 _load_state_dict_pre_hooks
│      │
│      └── 2c. 递归：对每个子模块，过滤出对应的 key，调用 load(child, child_prefix)
│
├── 3. strict=True 且有 mismatch → raise RuntimeError
├── 4. strict=False → 返回 _IncompatibleKeys(missing_keys, unexpected_keys)
└── 5. 返回 _IncompatibleKeys（namedtuple）

=== _load_from_state_dict 源码 ===
    def _load_from_state_dict(
        self,
        state_dict,
        prefix,
        local_metadata,
        strict,
        missing_keys,
        unexpected_keys,
        error_msgs,
    ):
        r"""Copy parameters and buffers from :attr:`state_dict` into only this module, but not its descendants.

        This is called on every submodule
        in :meth:`~torch.nn.Module.load_state_dict`. Metadata saved for this
        module in input :attr:`state_dict` is provided as :attr:`local_metadata`.
        For state dicts without metadata, :attr:`local_metadata` is empty.
        Subclasses can achieve class-specific backward compatible loading using
        the version number at `local_metadata.get("version", None)`.
        Additionally, :attr:`local_metadata` can also contain the key
        :attr:`_extra_state` which is a dict for the ``extra_state`` of this
        module.

        .. note::
            :attr:`state_dict` is not the same object as the input
            :attr:`state_dict` to :meth:`~torch.nn.Module.load_state_dict`. So
            it can be modified.

        Args:
            state_dict (dict): a dict containing parameters and
                persistent buffers.
            prefix (str): the prefix for parameters and buffers used in this
                module
            local_metadata (dict): a dict containing the metadata for this module.
                See
            strict (bool): whether to strictly enforce that the keys in
                :attr:`state_dict` with :attr:`prefix` match the names of
                parameters and buffers in this module
            missing_keys (list of str): if ``strict=True``, add missing keys to
                this list
            unexpected_keys (list of str): if ``strict=True``, add unexpected
                keys to this list
            error_msgs (list of str): error messages should be added to this
                list, and will be reported together in
                :meth:`~torch.nn.Module.load_state_dict`
        """
        for hook in self._load_state_dict_pre_hooks.values():
            hook(
                state_dict,
                prefix,
                local_metadata,
                strict,
                missing_keys,
                unexpected_keys,
                error_msgs,
            )

        persistent_buffers = {
            k: v
            for k, v in self._buffers.items()
            if k not in self._non_persistent_buffers_set
        }
        local_name_params = itertools.chain(
            self._parameters.items(), persistent_buffers.items()
        )
        local_state = {k: v for k, v in local_name_params if v is not None}
        load_local_state = (
            _local_load_assign
            if assign
            else _local_load_copy
        )

        for name, param in local_state.items():
            key = prefix + name
            if key in state_dict:
                input_param = state_dict[key]

                if not torch.overrides.is_tensor_like(input_param):
                    error_msgs.append(
                        f'While copying the parameter named "{key}", '
                        "expected torch.Tensor or Tensor-like object from checkpoint but "
                        f"received {type(input_param)}"
                    )
                    continue

                # 跳过 require_backend_checks（向后兼容）
                # ...

                # This is used to handle the backward-compatible case
                # when users set the flag in old versions' state dict.
                # ...

                try:
                    with torch.no_grad():
                        load_local_state(param, input_param)
                except Exception as ex:
                    error_msgs.append(
                        f'While copying the parameter named "{key}", '
                        f"expected tensors with size {param.size()}, "
                        f"dtype {param.dtype}, and device {param.device} "
                        f"but got size {input_param.size()}, "
                        f"dtype {input_param.dtype}, "
                        f"and device {input_param.device}."
                    )
                    # 保存原始异常链
                    error_msgs[-1] += f"\nThe above exception was the direct cause of the following exception:\n\n{type(ex).__name__}: {ex}"
            elif strict:
                missing_keys.append(key)

        extra_state_key = prefix + _EXTRA_STATE_KEY_SUFFIX
        if (
            getattr(self.__class__, "set_extra_state", nn.Module.set_extra_state)
            is not nn.Module.set_extra_state
        ):
            # 用户覆写了 set_extra_state
            if extra_state_key in state_dict:
                self.set_extra_state(state_dict[extra_state_key])
            elif strict:
                missing_keys.append(extra_state_key)
        elif extra_state_key in state_dict:
            # 默认 set_extra_state → 相当于 unexpected
            unexpected_keys.append(extra_state_key)

        if strict:
            for key in state_dict.keys():
                if key.startswith(prefix) and key != extra_state_key:
                    input_name = key[len(prefix) :]
                    input_name = input_name.split(".", 1)[0]
                    if (
                        input_name not in local_state
                        and input_name not in self._modules
                    ):
                        unexpected_keys.append(key)

_load_from_state_dict 逐 key 加载流程：

对于当前模块的每个参数和持久化 buffer：
│
├── key = prefix + name（如 "encoder.conv.weight"）
├── 如果 key 在 state_dict 中：
│     ├── 检查 input_param 是否是 Tensor-like
│     ├── load_local_state(param, input_param)  → copy_ 或 assign
│     └── 如果失败 → 记录 shape/dtype/device mismatch 错误
├── 如果 key 不在 state_dict 中 且 strict=True：
│     └── missing_keys.append(key)
│
├── 处理 extra_state_key
└── 检查 state_dict 中是否有 unexpected keys（属于当前 prefix 但不匹配任何参数/buffer/子模块）

    def set_extra_state(self, state: Any) -> None:

        This function is called from :func:`load_state_dict` to handle any
        extra state found within the `state_dict`. Implement this function and
        a corresponding :func:`get_extra_state` for your module if you need
        to store extra state within its `state_dict`.

        Args:
            state (dict): Extra state from the `state_dict`
        """

⚠️ 注意：默认实现会抛 RuntimeError。只有当你覆写了 set_extra_state，_load_from_state_dict 才会尝试从 state_dict 中加载 extra_state。这是一个设计细节 —— PyTorch 通过检查 set_extra_state 是否被覆写来决定行为。

extra_state 在 state_dict 中的 key 格式：{prefix}_extra_state（例如根模块的 key 是 _extra_state，子模块 encoder 是 encoder._extra_state）。

A.5 序列化 Hook 系统

module.py 在 __init__ 中预初始化了三组序列化 hook 字典：

Hook 字典 触发时机 注册方法
_state_dict_pre_hooks state_dict() 开始时 register_state_dict_pre_hook
_state_dict_hooks state_dict() 返回前 register_state_dict_post_hook
_load_state_dict_pre_hooks load_state_dict() 加载每个模块前 register_load_state_dict_pre_hook
_load_state_dict_post_hooks load_state_dict() 加载后 （内部使用）
💡 关键区别：_state_dict_pre_hooks 在 _save_to_state_dict 之前执行（可以修改模块状态），_state_dict_hooks 在 _save_to_state_dict 之后执行（可以修改 destination dict）。而 _load_state_dict_pre_hooks 在 _load_from_state_dict 之前执行（常用于版本兼容性处理）。
In [8]:

    def register_state_dict_pre_hook(self, hook):
        r"""Register a pre-hook for the :func:`~torch.nn.Module.state_dict` method.

        These hooks will be called with arguments: ``self``, ``prefix``,
        and ``keep_vars`` before calling ``state_dict`` on ``self``. The registered
        hooks can be used to pre-process the module's state before the
        ``state_dict`` call is made.

        The hook should have the following signature::

            hook(module, prefix, keep_vars) -> None

        If the hook returns something, it is ignored.
        """
        handle = hooks.RemovableHandle(self._state_dict_pre_hooks)
        self._state_dict_pre_hooks[handle.id] = hook
        return handle


=== register_state_dict_post_hook 源码 ===
    def register_state_dict_post_hook(self, hook):
        r"""Register a post-hook for the :func:`~torch.nn.Module.state_dict` method.

        These hooks will be called with arguments: ``self``, ``state_dict``,
        ``prefix``, and ``local_metadata`` after the ``state_dict`` of ``self``
        has been populated. The registered hooks can be used to perform
        post-processing on the ``state_dict`` before it is returned. The hook
        should have the following signature::

            hook(module, state_dict, prefix, local_metadata) -> state_dict

        If the hook returns something, it is used as the new ``state_dict`` to
        be returned by :func:`~torch.nn.Module.state_dict`.
        """
        handle = hooks.RemovableHandle(self._state_dict_hooks)
        self._state_dict_hooks[handle.id] = hook
        return handle


=== register_load_state_dict_pre_hook 源码 ===
    def register_load_state_dict_pre_hook(self, hook):
        r"""Register a pre-hook for the :func:`~torch.nn.Module.load_state_dict` method.

        These hooks will be called with arguments: ``state_dict``, ``prefix``,
        ``local_metadata``, ``strict``, ``missing_keys``, ``unexpected_keys``,
        and ``error_msgs``, before the ``state_dict`` is loaded into this
        module. The registered hooks can be used to pre-process the
        ``state_dict``, for instance to perform backward compatibility
        adjustments.

        The hook should have the following signature::

            hook(module, state_dict, prefix, local_metadata, strict,
                 missing_keys, unexpected_keys, error_msgs) -> None

        If the hook returns something, it is ignored.
        """
        handle = hooks.RemovableHandle(self._load_state_dict_pre_hooks)
        self._load_state_dict_pre_hooks[handle.id] = hook
        return handle

Part B: 机制分析

B.1 state_dict 的递归收集机制

state_dict() 采用双层递归设计：

  1. 叶子层（_save_to_state_dict）：收集当前模块的参数和 persistent buffer
  2. 递归层（module.state_dict(destination, prefix + name + ".", ...)）：对每个子模块递归调用
Model (prefix="")
├── _save_to_state_dict(dest, "", keep_vars)    → 保存 root param
├── conv (prefix="conv.")
│   ├── _save_to_state_dict(dest, "conv.", ...) → 保存 conv.weight, conv.bias
│   └── （conv 无子模块，递归结束）
├── bn (prefix="bn.")
│   ├── _save_to_state_dict(dest, "bn.", ...)   → 保存 bn.weight, bn.bias, bn.running_mean, bn.running_var
│   └── （bn 无子模块，递归结束）
└── fc (prefix="fc.")
    ├── _save_to_state_dict(dest, "fc.", ...)   → 保存 fc.weight, fc.bias
    └── （fc 无子模块，递归结束）

最终 destination 包含：

{
    "conv.weight": ..., "conv.bias": ...,
    "bn.weight": ..., "bn.bias": ..., "bn.running_mean": ..., "bn.running_var": ...,
    "fc.weight": ..., "fc.bias": ...,
    "_extra_state": ...  # 如果覆写了 get_extra_state
}

B.2 同名检查：点分隔路径如何避免歧义

PyTorch 通过前缀拼接机制保证了 key 的唯一性。但存在一个潜在歧义：

# 模块名 "layer.1" 包含点号 vs 嵌套结构 "layer" → "1"
# add_module 禁止点号出现在模块名中，从而消除了这个问题

add_module 中有明确的检查：elif "." in name: raise KeyError(...)，所以模块名中不可能出现 .，点号仅用于路径分隔符。

B.3 strict=True vs strict=False

特性 strict=True（默认） strict=False
missing_keys 存在即报错（RuntimeError） 返回 _IncompatibleKeys 对象
unexpected_keys 存在即报错 返回 _IncompatibleKeys 对象
典型场景 保存/加载同一个模型 迁移学习、微调（只加载部分层）
安全性 高，防止遗漏 低，需要手动检查 mismatch

迁移学习场景：预训练模型比目标模型多了一个 classifier 层，需要 strict=False：

pretrained_dict = torch.load('pretrained.pt')
model_dict = model.state_dict()
# 只加载匹配的 key
pretrained_dict = {k: v for k, v in pretrained_dict.items()
                   if k in model_dict and v.shape == model_dict[k].shape}
model.load_state_dict(pretrained_dict, strict=False)

B.4 extra_state 的设计模式

get_extra_state / set_extra_state 采用 Template Method 模式：

  • _save_to_state_dict 调用 get_extra_state() → 默认抛 RuntimeError
  • _load_from_state_dict 调用 set_extra_state() → 默认抛 RuntimeError
  • 用户覆写这两个方法来自定义行为
  • PyTorch 通过 getattr(self.__class__, "set_extra_state", ...) is not nn.Module.set_extra_state 检测是否被覆写

这个设计的巧妙之处：如果用户没有覆写，_load_from_state_dict 会将 state_dict 中的 _extra_state key 视为 unexpected（而不是报错），保持了向后兼容。

import time

class TimestampModule(nn.Module):
    def __init__(self):
        super().__init__()
        self.linear = nn.Linear(10, 10)
        # 一个普通 Python 属性，不会被 state_dict 自动保存
        self.last_saved = None

model = TimestampModule()

# --- Hook 1: state_dict_pre_hook — 保存前记录时间戳 ---
def pre_save_hook(module, prefix, keep_vars):
    """在 state_dict 开始前执行，参数和 buffer 还未被收集"""
    module.last_saved = time.time()
    print(f"[pre_hook] prefix='{prefix}', 记录保存时间: {module.last_saved:.2f}")

pre_handle = model.register_state_dict_pre_hook(pre_save_hook)

# --- Hook 2: state_dict_post_hook — 保存后向结果注入自定义数据 ---
def post_save_hook(module, state_dict, prefix, local_metadata):
    """在 state_dict 收集完成后执行，可以修改返回的 dict"""
    state_dict[f'{prefix}_saved_at'] = torch.tensor([module.last_saved])
    print(f"[post_hook] prefix='{prefix}', 注入了 _saved_at 字段")
    return state_dict  # 返回修改后的 state_dict

post_handle = model.register_state_dict_post_hook(post_save_hook)

# 执行 state_dict
print("=== 调用 state_dict() ===\n")
sd = model.state_dict()

print(f"\n=== 结果 ===")
for k, v in sd.items():
    print(f"  {k}: {v}")

print(f"\nmodel.last_saved: {model.last_saved:.2f}")

# 清理 hook
pre_handle.remove()
post_handle.remove()
print("\n✅ Hooks 已清理")

class StatefulModule(nn.Module):
    """带自定义状态的 Module —— 记录训练步数和配置"""
    def __init__(self, in_features, out_features, config_name=""):
        super().__init__()
        self.linear = nn.Linear(in_features, out_features)
        # 非 Tensor 状态：不会被 state_dict() 自动保存
        self.train_steps = 0
        self.config_name = config_name

    def forward(self, x):
        self.train_steps += 1  # 每次 forward 计数
        return self.linear(x)

    # --- 覆写 get_extra_state ---
    def get_extra_state(self):
        """保存自定义 Python 对象到 state_dict"""
        return {
            'train_steps': self.train_steps,
            'config_name': self.config_name,
        }

    # --- 覆写 set_extra_state ---
    def set_extra_state(self, state):
        """从 state_dict 恢复自定义状态"""
        self.train_steps = state.get('train_steps', 0)
        self.config_name = state.get('config_name', '')
        print(f"[set_extra_state] 恢复了 train_steps={self.train_steps}, config_name='{self.config_name}'")

    def __init__(self):
        super().__init__()

        self.head = nn.Linear(50, 10)  # v2 改名了！v1 叫 "fc"

    ('backbone.bias', torch.randn(50)),
    ('fc.weight', torch.randn(10, 50)),       # v1 的旧名字
    ('fc.bias', torch.randn(10)),
])

# --- 定义 pre-hook：自动重命名 ---
def rename_hook(module, state_dict, prefix, local_metadata,
                strict, missing_keys, unexpected_keys, error_msgs):
    """将 state_dict 中的 'fc.*' 重命名为 'head.*'"""
    keys_to_rename = [k for k in state_dict.keys() if k.startswith(prefix + 'fc.')]
    for old_key in keys_to_rename:
        new_key = old_key.replace(prefix + 'fc.', prefix + 'head.')
        if new_key not in state_dict:
            state_dict[new_key] = state_dict.pop(old_key)
            print(f"[pre_hook] 重命名: {old_key} → {new_key}")

model = ModelV2()

# 不注册 hook → strict=True 应该失败
print("=== 无 hook，strict=True 加载 ===")
try:
    model.load_state_dict(model_v1_sd, strict=True)
except RuntimeError as e:
    print(f"❌ 失败: {str(e)[:150]}...")

# 注册 hook 后再试
print("\n=== 注册 rename_hook 后，strict=True 加载 ===")
model2 = ModelV2()
handle = model2.register_load_state_dict_pre_hook(rename_hook)
result = model2.load_state_dict(model_v1_sd, strict=True)
print(f"✅ 成功! missing={result.missing_keys}, unexpected={result.unexpected_keys}")
print(f"head.weight 已加载: {model2.head.weight.shape}")

handle.remove()

实验 6：_IncompatibleKeys 的实用模式

编写一个健壮的 safe_load_state_dict，自动处理 shape mismatch 和 key mismatch。

In [17]:
def safe_load_state_dict(model, state_dict, verbose=True):
    """
    安全的 state_dict 加载器：
    1. 自动跳过 shape 不匹配的 key
    2. 自动跳过不存在的 key
    3. 报告所有 mismatch
    """
    model_sd = model.state_dict()

    # 过滤：只保留在模型中存在且 shape 匹配的 key
    filtered_sd = OrderedDict()
    skipped_shape = []
    skipped_missing = []

    for k, v in state_dict.items():
        if k in model_sd:
            if model_sd[k].shape == v.shape:
                filtered_sd[k] = v
            else:
                skipped_shape.append((k, model_sd[k].shape, v.shape))
        else:
            skipped_missing.append(k)

    # 加载过滤后的 state_dict
    result = model.load_state_dict(filtered_sd, strict=False)

    # 报告
    if verbose:
        print(f"=== 加载报告 ===")
        print(f"state_dict 总 key 数: {len(state_dict)}")
        print(f"成功加载: {len(filtered_sd)}")
        print(f"shape 不匹配（跳过）: {len(skipped_shape)}")
        for k, ms, ss in skipped_shape:
            print(f"  ⚠️  {k}: model shape={list(ms)}, checkpoint shape={list(ss)}")
        print(f"不存在（跳过）: {len(skipped_missing)}")
        for k in skipped_missing:
            print(f"  ⚠️  {k}")
        print(f"模型中有但 state_dict 没有: {len(result.missing_keys)}")
        for k in result.missing_keys:
            print(f"  ⚠️  {k}")

    return {
        'loaded': len(filtered_sd),
        'skipped_shape': skipped_shape,
        'skipped_missing': skipped_missing,
        'missing_in_sd': result.missing_keys,
    }

Part D: 小结

核心方法调用链

保存路径：
  model.state_dict()
    ├── _state_dict_pre_hooks（可修改模块状态）
    ├── _save_to_state_dict(dest, prefix, keep_vars)
    │     ├── 遍历 _parameters → dest[prefix + name] = param.detach()
    │     ├── 遍历 _buffers（跳过 non-persistent）→ dest[prefix + name] = buf.detach()
    │     └── 调用 get_extra_state() → dest[prefix + _EXTRA_STATE_KEY_SUFFIX]
    ├── 递归：child.state_dict(destination, prefix + name + ".", keep_vars)
    └── _state_dict_hooks（可修改 destination）

加载路径：
  model.load_state_dict(state_dict, strict=True)
    ├── load(module, prefix)
    │     ├── _load_from_state_dict()
    │     │     ├── _load_state_dict_pre_hooks（可修改 state_dict 做兼容处理）
    │     │     ├── 对每个 local param/buffer：copy_ 或 assign
    │     │     ├── 处理 extra_state
    │     │     └── 检查 missing_keys / unexpected_keys
    │     └── 递归：load(child, child_prefix)
    └── strict=True 且有 mismatch → RuntimeError
        strict=False → 返回 _IncompatibleKeys(missing_keys, unexpected_keys)

要点清单

  • state_dict() 递归收集所有参数和 persistent buffer，key 格式为 {prefix}{name}
  • _save_to_state_dict 只收集当前模块的状态（不递归子模块）
  • non_persistent_buffers_set 中的 buffer 不会出现在 state_dict 中
  • load_state_dict(strict=True) 要求 key 完全匹配，否则抛出 RuntimeError
  • load_state_dict(strict=False) 返回 _IncompatibleKeys 对象，适合迁移学习
  • register_state_dict_pre_hook 在保存前执行（如记录时间戳）
  • register_state_dict_post_hook 在保存后执行（如注入自定义 key）
  • register_load_state_dict_pre_hook 在加载前执行（如重命名旧 key → 新 key）
  • get_extra_state / set_extra_state 用于保存/加载非 Tensor 数据
  • _IncompatibleKeys 是 namedtuple，包含 missing_keys 和 unexpected_keys

常见陷阱

陷阱 说明
strict=True 迁移学习 直接加载预训练模型到不同结构会报错，需 strict=False
extra_state 默认抛异常 必须覆写 get_extra_state / set_extra_state，否则保存/加载会失败
shape 不匹配不会自动跳过 strict=False 下，shape 不匹配会导致 copy_ 失败（报错）而不是跳过
hook 的返回值 state_dict_post_hook 的返回值会替换 destination，不返回则使用原值
non-persistent buffer 如 Dropout 的 mask，不会保存。加载后需要重新生成
