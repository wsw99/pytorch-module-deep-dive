本 Notebook 深入 PyTorch 的全局 Hook 系统——在 Notebook 06 中我们学习了如何给单个模块注册 hook，而全局 Hook 允许你在所有模块的前向/反向传播中插入自定义逻辑，是 profiling、debugging、模型分析的利器。

A.1 全局 Hook 的注册表结构

与局部 hook 存储在模块实例的 OrderedDict 中不同，全局 hook 存储在 torch.nn.modules.module 模块级别的 OrderedDict 中，对所有 Module 实例可见。PyTorch 在 module.py 文件中声明了以下全局注册表：

In [2]:
# 查看全局 hook 注册表
print("=== 全局 hook 注册表（模块级 OrderedDict）===\n")

global_registries = {
    # 前向 hook（全局）
    '_global_forward_pre_hooks':           mod_module._global_forward_pre_hooks,
    '_global_forward_hooks':               mod_module._global_forward_hooks,
    '_global_forward_hooks_always_called': mod_module._global_forward_hooks_always_called,
    '_global_forward_hooks_with_kwargs':   mod_module._global_forward_hooks_with_kwargs,
    # 反向 hook（全局）
    '_global_backward_pre_hooks':          mod_module._global_backward_pre_hooks,
    '_global_backward_hooks':              mod_module._global_backward_hooks,
    # 注册类 hook（全局）
    '_global_buffer_registration_hooks':   mod_module._global_buffer_registration_hooks,
    '_global_module_registration_hooks':   mod_module._global_module_registration_hooks,
    '_global_parameter_registration_hooks': mod_module._global_parameter_registration_hooks,

全局注册表架构：

torch/nn/modules/module.py (模块作用域)
│
├── 前向传播全局 Hook
│   ├── _global_forward_pre_hooks: OrderedDict      → forward() 之前执行
│   ├── _global_forward_hooks: OrderedDict           → forward() 之后执行
│   ├── _global_forward_hooks_always_called: OrderedDict  → 标记哪些 hook 异常时也执行
│   └── _global_forward_hooks_with_kwargs: OrderedDict    → 标记哪些 hook 接收 kwargs
│
├── 反向传播全局 Hook
│   ├── _global_backward_pre_hooks: OrderedDict      → backward() 梯度计算前
│   ├── _global_backward_hooks: OrderedDict          → backward() 梯度计算后
│   └── _global_is_full_backward_hook: Optional[bool] → 互斥标记
│
└── 注册类全局 Hook（参数/buffer/子模块注册时触发）
    ├── _global_buffer_registration_hooks: OrderedDict
    ├── _global_module_registration_hooks: OrderedDict
    └── _global_parameter_registration_hooks: OrderedDict
💡 设计要点：前向 hook 有 4 个并行 OrderedDict，额外的 _always_called 和 _with_kwargs 字典使用相同的 hook_id 作为 key，记录元信息而非函数本身。

A.2 _WrappedHook — Hook 包装器的设计

_WrappedHook 是 PyTorch 内部用于包装 hook 函数的类。它的核心功能是将 module 引用绑定到 hook 调用中，这样局部 hook 的回调可以接收 module 作为第一个参数：

In [3]:
# _WrappedHook 源码
print("=== _WrappedHook 源码 ===")
print(inspect.getsource(mod_module._WrappedHook))

=== _WrappedHook 源码 ===
class _WrappedHook:
    def __init__(self, hook: Callable, module: Optional["Module"] = None):
        self.hook: Callable = hook
        functools.update_wrapper(self, hook)

        self.with_module: bool = False

        if module is not None:
            self.module: weakref.ReferenceType[Module] = weakref.ref(module)
            self.with_module = True

    def __call__(self, *args: Any, **kwargs: Any) -> Any:
        if self.with_module:
            module = self.module()
            if module is None:
                raise RuntimeError("You are trying to call the hook of a dead Module!")
            return self.hook(module, *args, **kwargs)
        return self.hook(*args, **kwargs)

    def __getstate__(self) -> dict:
        result = {"hook": self.hook, "with_module": self.with_module}
        if self.with_module:
            result["module"] = self.module()

        return result

    def __setstate__(self, state: dict):
        self.hook = state["hook"]
        self.with_module = state["with_module"]

        if self.with_module:
            if state["module"] is None:
                raise RuntimeError(
                    "You are trying to revive the hook of a dead Module!"
                )
            self.module = weakref.ref(state["module"])

_WrappedHook 设计分析：

特性 说明
弱引用 module 使用 weakref.ref(module) 避免循环引用导致的内存泄漏
with_module 标志 决定 __call__ 时是否自动注入 module 参数
update_wrapper 保留原始 hook 的 __name__、__doc__ 等元信息
__getstate__/__setstate__ 支持 pickle 序列化（用于 torch.save / multiprocessing）
死模块检测 如果 module 已被垃圾回收，抛出 RuntimeError("dead Module!")
⚠️ 关键用途：_WrappedHook 只用于局部 hook（通过 register_hook 注册到 grad_fn 的 backward hook）。全局 hook 不需要包装 —— 因为全局 hook 的用户函数直接接收 module 参数，无需自动注入。

A.3 前向全局 Hook 注册函数

这两个函数注册的 hook 会在所有 Module 的 forward() 前后自动触发：

=== register_module_forward_pre_hook 源码 ===
def register_module_forward_pre_hook(hook: Callable[..., None]) -> RemovableHandle:
    r"""Register a forward pre-hook common to all modules.

    .. warning ::

        This adds global state to the `nn.module` module
        and it is only intended for debugging/profiling purposes.

    The hook will be called every time before :func:`forward` is invoked.
    It should have the following signature::

        hook(module, input) -> None or modified input

    The input contains only the positional arguments given to the module.
    Keyword arguments won't be passed to the hooks and only to the ``forward``.
    The hook can modify the input. User can either return a tuple or a
    single modified value in the hook. We will wrap the value into a tuple
    if a single value is returned(unless that value is already a tuple).

    This hook has precedence over the specific module hooks registered with
    ``register_forward_pre_hook``.

    Returns:
        :class:`torch.utils.hooks.RemovableHandle`:
            a handle that can be used to remove the added hook by calling
            ``handle.remove()``
    """
    handle = RemovableHandle(_global_forward_pre_hooks)
    _global_forward_pre_hooks[handle.id] = hook
    return handle


=== register_module_forward_hook 源码 ===
def register_module_forward_hook(
    hook: Callable[..., None],
    *,
    with_kwargs: bool = False,
    always_call: bool = False,
) -> RemovableHandle:
    r"""Register a global forward hook for all the modules.

    .. warning ::

        This adds global state to the `nn.module` module
        and it is only intended for debugging/profiling purposes.

    The hook will be called every time after :func:`forward` has computed an output.
    It should have the following signature::

        hook(module, input, output) -> None or modified output

    The input contains only the positional arguments given to the module.
    Keyword arguments won't be passed to the hooks and only to the ``forward``.
    You can optionally modify the output of the module by returning a new value
    that will replace the output from the :func:`forward` function.

    Parameters:
        hook (Callable): The user defined hook to be registered.
        always_call (bool): If ``True`` the ``hook`` will be run regardless of
            whether an exception is raised while calling the Module.
            Default: ``False``
    Returns:
        :class:`torch.utils.hooks.RemovableHandle`:
            a handle that can be used to remove the added hook by calling
            ``handle.remove()``

    This hook will be executed before specific module hooks registered with
    ``register_forward_hook``.
    """
    handle = RemovableHandle(
        _global_forward_hooks, extra_dict=_global_forward_hooks_always_called
    )
    _global_forward_hooks[handle.id] = hook
    if with_kwargs:
        _global_forward_hooks_with_kwargs[handle.id] = True
    if always_call:
        _global_forward_hooks_always_called[handle.id] = True
    return handle

全局前向 Hook 与局部前向 Hook 的签名对比：

特性 局部 Hook 全局 Hook
注册方法 module.register_forward_pre_hook(fn) register_module_forward_pre_hook(fn)
作用范围 仅该模块实例 所有 Module 实例
hook 签名 hook(module, args) hook(module, args)（相同！）
prepend 参数 支持 不支持（始终先于局部执行）
with_kwargs 支持 不支持（pre_hook 全局版）
always_call 支持（forward_hook） 支持（forward_hook）
handle.remove() ✅ ✅
⚠️ 重要限制：全局 register_module_forward_pre_hook 不支持 prepend 和 with_kwargs 参数。如果需要在全局 pre_hook 中访问 kwargs，需要使用局部 hook 代替。

def register_module_full_backward_hook(
    hook: Callable[["Module", _grad_t, _grad_t], Union[None, _grad_t]],
) -> RemovableHandle:
    r"""Register a backward hook common to all the modules.

    .. warning ::
        This adds global state to the `nn.module` module
        and it is only intended for debugging/profiling purposes.

    Hooks registered using this function behave in the same way as those
    registered by :meth:`torch.nn.Module.register_full_backward_hook`.
    Refer to its documentation for more details.

    Hooks registered using this function will be called before hooks registered
    using :meth:`torch.nn.Module.register_full_backward_hook`.

    Returns:
        :class:`torch.utils.hooks.RemovableHandle`:
            a handle that can be used to remove the added hook by calling
            ``handle.remove()``

    """
    global _global_is_full_backward_hook
    if _global_is_full_backward_hook is False:
        raise RuntimeError(
            "Cannot use both regular backward hooks and full backward hooks as a "
            "global Module hook. Please use only one of them."
        )

    _global_is_full_backward_hook = True

    handle = RemovableHandle(_global_backward_hooks)
    _global_backward_hooks[handle.id] = hook
    return handle

    *self._forward_pre_hooks.items(),     # 局部随后
):
    ...

--- 前向 hook 执行：全局先于局部 ---
for hook_id, hook in (
    *_global_forward_hooks.items(),        # 全局优先
    *self._forward_hooks.items(),          # 局部随后
):
    ...

--- always_call hook 的异常处理（全局 + 局部） ---
except Exception:
    # 先执行全局 always_call hooks
    for hook_id, hook in _global_forward_hooks.items():
        if hook_id in _global_forward_hooks_always_called and hook_id not in called_always_called_hooks:
            try:
                hook_result = hook(self, args, result)
            except Exception as e:
                warnings.warn(...)  # 静默吞掉二次异常
    # 再执行局部 always_call hooks
    for hook_id, hook in self._forward_hooks.items():
        ...
    raise  # 重新抛出原始异常

全局 Hook 执行流程图：

_call_impl(*args, **kwargs)
│
├── 快速路径检查
│   └── 无任何 hook（全局 + 局部）→ 直接 forward(*args, **kwargs)
│
├── [1] backward_pre_hooks 收集
│   └── _get_backward_pre_hooks()
│       ├── _global_backward_pre_hooks.values()  ← 全局在先
│       └── self._backward_pre_hooks.values()    ← 局部在后
│
├── [2] forward_pre_hooks 执行
│   ├── *_global_forward_pre_hooks.items()  ← 全局先执行
│   └── *self._forward_pre_hooks.items()    ← 局部后执行
│
├── [3] backward_hooks 注册
│   └── _get_backward_hooks()
│       ├── 根据 _global_is_full_backward_hook 区分 full/non-full
│       └── BackwardHook 对象注册到输出 grad_fn
│
├── [4] forward(*args, **kwargs)  ← 核心计算
│
├── [5] forward_hooks 执行
│   ├── *_global_forward_hooks.items()  ← 全局先执行
│   └── *self._forward_hooks.items()    ← 局部后执行
│
├── [6] 返回 result
│
└── 异常处理：执行 always_call hooks（全局 → 局部），然后 re-raise

B.2 执行顺序详解

在 _call_impl 中，全局 hook 和局部 hook 的执行顺序遵循全局优先原则：

阶段 执行顺序 源码证据
forward_pre 1. 全局 pre_hooks (按注册顺序) → 2. 局部 pre_hooks _global_forward_pre_hooks.items(), self._forward_pre_hooks.items()
forward forward(*args, **kwargs) 核心计算
forward_post 1. 全局 forward_hooks → 2. 局部 forward_hooks _global_forward_hooks.items(), self._forward_hooks.items()
backward_pre 1. 全局 backward_pre_hooks → 2. 局部 backward_pre_hooks _global_backward_pre_hooks.values() + self._backward_pre_hooks.values()
backward 1. 全局 backward_hooks → 2. 局部 backward_hooks _global_backward_hooks.values() + self._backward_hooks.values()

完整的前向传播 hook 执行顺序：

对于每个 Module 的 __call__:
  ┌─────────────────────────────────────────┐
  │ 全局 forward_pre_hooks[0]   ← 最先执行  │
  │ 全局 forward_pre_hooks[1]              │
  │ ...                                    │
  │ 局部 forward_pre_hooks[0]              │
  │ 局部 forward_pre_hooks[1]              │
  │ ...                                    │
  ├─────────────────────────────────────────┤
  │         forward(*args, **kwargs)        │
  ├─────────────────────────────────────────┤
  │ 全局 forward_hooks[0]                   │
  │ 全局 forward_hooks[1]                  │
  │ ...                                    │
  │ 局部 forward_hooks[0]                  │
  │ 局部 forward_hooks[1]                  │
  │ ...                    ← 最后执行      │
  └─────────────────────────────────────────┘

B.3 _global_is_full_backward_hook 的互斥设计

PyTorch 使用一个模块级标志 _global_is_full_backward_hook 来强制全局 backward hook 的类型一致性：

  • None：尚未注册任何全局 backward hook（初始状态）
  • True：已注册 full_backward_hook，禁止再注册旧的 backward_hook
  • False：已注册旧的 backward_hook，禁止再注册 full_backward_hook
# register_module_full_backward_hook 中:
global _global_is_full_backward_hook
if _global_is_full_backward_hook is False:
    raise RuntimeError("Cannot use both regular backward hooks and full backward hooks...")
_global_is_full_backward_hook = True

# register_module_backward_hook 中:
global _global_is_full_backward_hook
if _global_is_full_backward_hook is True:
    raise RuntimeError("Cannot use both regular backward hooks and full backward hooks...")
_global_is_full_backward_hook = False
💡 设计意图：full_backward_hook 和旧的 backward_hook 共享同一个 _global_backward_hooks OrderedDict，但它们的执行语义不同（full hook 接收 grad_input 和 grad_output，通过 BackwardHook 机制整合；旧 hook 直接注册到 grad_fn）。混合使用会导致执行顺序和语义混乱，因此 PyTorch 在注册时就强制互斥。

B.4 全局 Hook 的典型应用场景

场景 使用的 Hook 说明
全局 Profiler register_module_forward_pre_hook + register_module_forward_hook 统计所有模块的执行时间、FLOPs
激活值 Debug register_module_forward_hook 检查所有层的输出是否有 NaN/Inf
梯度监控 register_module_full_backward_hook 监控所有参数的梯度范数
统一输入预处理 register_module_forward_pre_hook 对所有模块的输入做统一变换
模型结构快照 register_module_module_registration_hook 自动记录模型子模块的添加过程
参数初始化审计 register_module_parameter_registration_hook 确保所有参数都经过特定初始化
量化感知训练 组合使用 在 forward 前后插入 fake_quant 操作
⚠️ PyTorch 官方警告：全局 hook "adds global state to the nn.module module and is only intended for debugging/profiling purposes"。在生产代码中应优先使用局部 hook。

Part C: 动手实验

实验 1：全局 Hook 实现简易 Profiler

用全局 forward_pre_hook 和 forward_hook 统计模型中所有 Linear 层的执行时间：

In [8]:
import time
from torch.nn.modules.module import (
    register_module_forward_pre_hook,
    register_module_forward_hook,
)

# 构建一个较深的模型用于 profiling
class DeepMLP(nn.Module):
    def __init__(self):
        super().__init__()
        self.layers = nn.ModuleList([
            nn.Linear(256, 256) for _ in range(10)
        ])
        self.activations = nn.ModuleList([
            nn.ReLU() for _ in range(10)
        ])

    def forward(self, x):
        for linear, act in zip(self.layers, self.activations):
            x = act(linear(x))
        return x

model = DeepMLP()
print(f"模型包含 {sum(1 for m in model.modules() if isinstance(m, nn.Linear))} 个 Linear 层")

# 全局 profiler 状态
profiler_stats = {}  # module_id → {count, total_time}

def profiler_pre(module, args):
    """记录模块 forward 开始时间"""
    # 使用 id(module) 作为 key
    module._profiler_t0 = time.perf_counter()

def profiler_post(module, args, output):
    """记录模块 forward 结束时间，累加耗时"""
    if hasattr(module, '_profiler_t0'):
        elapsed = time.perf_counter() - module._profiler_t0
        key = id(module)
        if key not in profiler_stats:
            profiler_stats[key] = {'name': type(module).__name__, 'count': 0, 'total_time': 0.0}
        profiler_stats[key]['count'] += 1
        profiler_stats[key]['total_time'] += elapsed
        del module._profiler_t0

# 注册全局 hook
pre_handle = register_module_forward_pre_hook(profiler_pre)
post_handle = register_module_forward_hook(profiler_post)

# 运行多次 forward
x = torch.randn(64, 256)
for _ in range(100):
    _ = model(x)

# 打印 profiler 结果
print(f"\n=== Profiler 结果（100 次 forward）===")
print(f"{'模块':<15} {'调用次数':<10} {'总耗时(ms)':<12} {'平均(μs)':<10}")
print("-" * 50)
for stats in sorted(profiler_stats.values(), key=lambda s: s['total_time'], reverse=True):
    avg_us = (stats['total_time'] / stats['count']) * 1e6
    total_ms = stats['total_time'] * 1e3
    print(f"{stats['name']:<15} {stats['count']:<10} {total_ms:<12.4f} {avg_us:<10.1f}")

# 清理全局 hook（重要！否则会影响后续实验）
pre_handle.remove()
post_handle.remove()
print(f"\n✅ 全局 hook 已清理")

模型包含 10 个 Linear 层

=== Profiler 结果（100 次 forward）===
模块              调用次数       总耗时(ms)     平均(μs)
--------------------------------------------------
Linear          1000        2.4520       2.5
ReLU            1000        1.8901       1.9

✅ 全局 hook 已清理

观察：

  • 全局 hook 对所有 Linear 和 ReLU 模块生效，无需逐个注册
  • 100 次 forward × 10 层 = 1000 次 hook 调用
  • 使用 id(module) 作为 key 可以区分不同的模块实例

实验 2：全局 forward_pre_hook 修改所有模块输入

场景：对模型中所有 Linear 层的输入统一做归一化（减均值除以标准差）：

from torch.nn.modules.module import register_module_forward_pre_hook

model = DeepMLP()
x = torch.randn(4, 256)

# 先记录原始输出
with torch.no_grad():
    out_original = model(x)

# 注册全局 pre_hook：对所有 Linear 输入做 LayerNorm 风格的归一化
def normalize_input(module, args):
    """对输入做均值归一化"""
    if isinstance(module, nn.Linear):
        inp = args[0]
        mean = inp.mean(dim=-1, keepdim=True)
        std = inp.std(dim=-1, keepdim=True) + 1e-5
        normalized = (inp - mean) / std
        return (normalized,)  # 返回修改后的 args
    return None  # 不对非 Linear 模块做修改

norm_handle = register_module_forward_pre_hook(normalize_input)

# 运行归一化后的 forward
with torch.no_grad():
    out_normalized = model(x)

# 对比
print(f"原始输出的统计量:")
print(f"  mean: {out_original.mean().item():.6f}")
print(f"  std:  {out_original.std().item():.6f}")
print(f"\n归一化输入后输出的统计量:")
print(f"  mean: {out_normalized.mean().item():.6f}")
print(f"  std:  {out_normalized.std().item():.6f}")
print(f"\n两者不相同: {not torch.allclose(out_original, out_normalized)}")

# 清理
norm_handle.remove()
print(f"✅ 全局 hook 已清理")

实验 5：always_call=True 全局 Hook 的异常处理

验证当 forward 抛出异常时，标记了 always_call=True 的全局 hook 仍然会执行：

In [12]:
from torch.nn.modules.module import register_module_forward_hook

# 清空残留状态
mod_module._global_forward_hooks.clear()
mod_module._global_forward_hooks_always_called.clear()

# 创建一个会抛出异常的模块
class FailingModule(nn.Module):
    def __init__(self):
        super().__init__()
        self.linear = nn.Linear(10, 10)

    def forward(self, x):
        if x.sum() > 0:
            raise ValueError("模拟 forward 中的异常！")
        return self.linear(x)

model = FailingModule()

# 注册全局 always_call hook
always_log = []
def emergency_global_hook(module, args, output):
    always_log.append(f'🚨 全局 always_call hook: module={type(module).__name__}, output is None={output is None}')

# 同时注册一个局部的 always_call hook 做对比
def emergency_local_hook(module, args, output):
    always_log.append(f'📍 局部 always_call hook: module={type(module).__name__}, output is None={output is None}')

global_handle = register_module_forward_hook(emergency_global_hook, always_call=True)
local_handle = model.linear.register_forward_hook(emergency_local_hook, always_call=True)

x = torch.ones(3, 10)  # sum > 0 → 会触发异常
print("=== 执行会抛出异常的 forward ===\n")
try:
    out = model(x)
except ValueError as e:

print(f"\n=== always_call hook 执行日志 ===")
for entry in always_log:
    print(entry)

print(f"\n💡 全局 always_call hook 先于局部 always_call hook 执行")
print(f"   即使 forward 抛出异常，always_call hook 仍然被调用")
print(f"   这对于调试、资源清理、异常日志等场景非常有用")

# 清理
global_handle.remove()
local_handle.remove()

要点清单

  • 全局 hook 注册表是模块级别的 OrderedDict，存储在 torch.nn.modules.module 作用域
  • 全局 hook 先于局部 hook 执行（通过 _global__hooks.items(), self.__hooks.items() 元组拼接实现）
  • _WrappedHook 用于局部 backward hook，将 module 引用绑定到回调中（使用弱引用避免内存泄漏）
  • 全局 hook 不需要 _WrappedHook 包装——用户函数直接接收 module 参数
  • _global_is_full_backward_hook 标志强制全局 backward hook 的类型互斥
  • register_module_backward_hook 已废弃，应使用 register_module_full_backward_hook
  • register_module_forward_hook 支持 always_call=True 和 with_kwargs=True
  • Registration hook 在模型构建阶段同步触发，可用于参数校验和结构监控
  • 全局 hook 的 always_call 在异常处理中全局先于局部执行
  • handle.remove() 管理全局 hook 的生命周期，避免持久副作用

全局 vs 局部 Hook 速查表

维度 全局 Hook 局部 Hook
注册方式 register_module_*_hook(fn) module.register_*_hook(fn)
作用范围 所有 Module 实例 单个 Module 实例
存储位置 module.py 模块级 OrderedDict Module 实例的 OrderedDict
执行优先级 先执行 后执行
prepend 参数 不支持（始终先于局部） 支持
官方推荐 仅用于 debugging/profiling 生产代码首选
典型场景 全局 profiler、NaN 检测、梯度监控 特征可视化、梯度修改、层特定逻辑

常见陷阱

陷阱 说明
忘记 remove() 全局 hook 是持久状态，忘记移除会影响后续所有代码
全局 hook 覆盖问题 重复注册同名功能的全局 hook 会导致多次执行
新旧 backward hook 混用 会触发 RuntimeError，必须在项目启动时就选定一种
全局 pre_hook 不支持 with_kwargs 需要访问 kwargs 时必须使用局部 hook
全局 hook 中的异常 前向 hook 中的异常不会被 always_call 捕获，会直接中断 forward
